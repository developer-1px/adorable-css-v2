Syntax Guide

AdorableCSS uses a unified, functional syntax. Learn the few rules you need to express any style.

1. The Functional Syntax

Everything is a function call. No more guessing if it's a dash, underscore, or camelCase.

Structure
rule(value)
rule: The property (e.g. `w`, `text`, `bg`)
value: The argument (e.g. `100px`, `red`, `bold`)
Examples
w(100px) → width: 100px
c(blue) → color: blue
grid(2) → grid-template-columns: repeat(2, 1fr)

2. Arguments & Composition

Pass multiple values using the slash / separator. No spaces allowed inside.

PatternUsageExample
/ SlashSeparate multiple argumentstext(xl/1.5) (size/leading)
. DotOpacity modifierbg(red.5) (50% opacity)
+ PlusCombine keywordshbox(top+center)
- MinusNegative numbersm(-20px)

3. Modifiers (States & Responsive)

Chain modifiers with a colon :. Order doesn't matter, but convention is breakpoint:state:rule.

States

  • hover:bg(red)
  • focus:ring(2)
  • active:scale(0.9)
  • disabled:opacity(0.5)

Breakpoints

  • sm:w(full)
  • md:hbox
  • lg:p(xl)
  • ..md:hidden (max-width)

Advanced

  • dark:c(white)
  • search:w(full) (container query)
  • !important (at end)
  • @print:hidden
